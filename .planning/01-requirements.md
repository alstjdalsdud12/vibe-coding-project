# 요구사항 명세

## 기능 요구사항 (Functional Requirements)

---

### FR-01: 캐릭터 생성 화면

| ID | 요구사항 |
|----|----------|
| FR-01-1 | 유저는 캐릭터의 외형/생김새를 자유 텍스트로 입력할 수 있다 (예: "검은 머리, 키 크고 우아함") |
| FR-01-2 | 유저는 무기를 선택하거나 직접 입력할 수 있다 (예: 검, 활, 지팡이) |
| FR-01-3 | 유저는 캐릭터 설정/컨셉을 자유 텍스트로 입력할 수 있다 (예: "신중하고 차분한 마법사") |
| FR-01-4 | 3가지 입력이 모두 채워진 경우에만 "캐릭터 생성" 버튼이 활성화된다 |
| FR-01-5 | "캐릭터 생성" 버튼을 누르면 AI 생성이 시작되고 로딩 인디케이터가 표시된다 |
| FR-01-6 | 로딩 중에는 입력 필드와 버튼이 비활성화된다 |

---

### FR-02: AI 캐릭터 생성 (Claude API)

| ID | 요구사항 |
|----|----------|
| FR-02-1 | Claude API는 유저 입력(외형, 무기, 컨셉)을 분석해 캐릭터 전체 정보를 생성한다 |
| FR-02-2 | **캐릭터명**: AI가 컨셉에 어울리는 고유한 이름을 자동 생성한다 |
| FR-02-3 | **스탯**: HP, 공격력(ATK), 방어력(DEF), 마나(MP) 수치를 생성한다 |
| FR-02-4 | 스탯은 컨셉과 무기에 연관성이 있어야 한다 (예: 마법사 → 높은 MP, 낮은 ATK) |
| FR-02-5 | **특수능력**: 컨셉에 맞는 고유 능력 1~3개를 생성한다 (이름 + 효과 설명) |
| FR-02-6 | **배경 스토리**: 캐릭터의 배경을 2~3문장으로 생성한다 |
| FR-02-7 | Claude 응답은 JSON 형식이어야 하며 파싱 가능해야 한다 |
| FR-02-8 | 생성 실패 시 오류 메시지를 유저에게 표시하고 재시도 버튼을 제공한다 |

**Claude 출력 JSON 스키마:**
```json
{
  "name": "실바나 아쉬크로프트",
  "stats": {
    "hp": 85,
    "atk": 35,
    "def": 25,
    "mp": 120
  },
  "abilities": [
    {
      "name": "정적의 화살",
      "description": "MP 20 소모. 대상에게 ATK×1.8 피해를 주고 1턴 동안 침묵 상태로 만든다."
    },
    {
      "name": "마나 방벽",
      "description": "MP 30 소모. 다음 공격을 완전히 무효화하는 방어막을 생성한다."
    }
  ],
  "story": "오래된 숲의 끝자락에서 홀로 수련한 마법사. 말보다 행동을 앞세우며, 감정을 드러내지 않지만 동료를 위해서라면 모든 것을 걸 수 있는 신념을 지니고 있다."
}
```

### FR-03: 캐릭터 저장 & 불러오기 (Firebase)

| ID | 요구사항 |
|----|----------|
| FR-03-1 | 생성된 캐릭터는 Firebase Firestore에 자동으로 저장된다 |
| FR-03-2 | 메인 메뉴에서 이전에 생성한 캐릭터 목록을 불러올 수 있다 |
| FR-03-3 | 목록에는 캐릭터명, 무기, 컨셉이 요약 표시된다 |
| FR-03-4 | 목록에서 캐릭터를 선택하면 상세 정보(스탯, 능력, 스토리)를 볼 수 있다 |
| FR-03-5 | 유저는 캐릭터를 삭제할 수 있다 (삭제 전 확인 다이얼로그 표시) |
| FR-03-6 | 저장된 캐릭터를 선택해 게임을 시작할 수 있다 |

---

### FR-04: 텍스트 기반 2D 게임플레이

| ID | 요구사항 |
|----|----------|
| FR-04-1 | 생성/선택된 캐릭터로 즉시 게임을 시작할 수 있다 |
| FR-04-2 | 전투는 텍스트 로그 방식으로 진행된다 (이미지 없음) |
| FR-04-3 | 전투 화면에는 나의 캐릭터 정보(이름, HP, MP)와 적 정보가 텍스트로 표시된다 |
| FR-04-4 | 유저는 매 턴 행동을 선택할 수 있다: 일반 공격 / 스킬 사용 / 도망 |
| FR-04-5 | 스킬 버튼은 캐릭터의 특수능력 개수만큼 표시되며 MP 소모량이 명시된다 |
| FR-04-6 | 전투 로그에는 행동 결과가 문장으로 출력된다 (예: "실바나가 정적의 화살을 쏜다! 45 피해!") |
| FR-04-7 | 데미지 계산: 기본 피해 = 공격자 ATK - 방어자 DEF (최소 1) |
| FR-04-8 | 캐릭터 또는 적의 HP가 0 이하가 되면 전투가 종료된다 |
| FR-04-9 | 승리 시 클리어 화면, 패배 시 게임 오버 화면이 표시된다 |
| FR-04-10 | 게임 오버/클리어 후 메인 메뉴로 돌아갈 수 있다 |

## 비기능 요구사항 (Non-Functional Requirements)

| ID | 항목 | 요구사항 |
|----|------|----------|
| NFR-01 | 응답 속도 | 캐릭터 생성 API 응답 30초 이내 |
| NFR-02 | 보안 | Claude API 키는 백엔드 서버에서만 사용, 클라이언트에 노출 금지 |
| NFR-03 | 보안 | Firebase 서비스 계정 키는 백엔드 환경 변수로만 관리 |
| NFR-04 | 플랫폼 | Android 10 이상 지원 (iOS는 선택) |
| NFR-05 | 해상도 | 360×640 ~ 1440×3200 모바일 해상도 대응 |
| NFR-06 | 오프라인 | 캐릭터 목록은 네트워크 없이도 Firebase 로컬 캐시로 조회 가능 |

## 제약 조건

- Claude API 비용: 개발 중에는 mock 응답 사용 권장
- Firebase Firestore: 무료 티어(Spark 플랜) 사용 가능 (읽기 50,000회/일, 쓰기 20,000회/일)
- 동시 사용자: MVP는 단일 사용자 기준 (계정 시스템 없음)
- 이미지 없음: 모든 게임 표현은 텍스트 UI로 처리
