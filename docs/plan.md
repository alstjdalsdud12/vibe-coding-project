# 기획서

## 1. 프로젝트 개요

| 항목 | 내용 |
|------|------|
| 프로젝트명 | AI 기반 캐릭터 생성 2D 모바일 게임 |
| 개발자 | choi (1인) |
| 개발 기간 | 7주 (Session 1 ~ Session 7) |

---

## 2. 프로젝트 목적 & 배경

### 목적
유저가 캐릭터의 외형·무기·컨셉을 텍스트로 입력하면 AI가 이름·스탯·특수능력·배경 스토리를 자동 생성하고, 생성된 캐릭터로 즉시 게임을 플레이할 수 있는 모바일 게임을 만든다.

### 배경
기존 RPG 게임은 개발자가 미리 만들어둔 캐릭터 중에서 고르는 방식이다. 이 프로젝트는 AI를 활용해 유저마다 다른 고유한 캐릭터를 자동 생성함으로써 개인화된 게임 경험을 제공한다.

### 핵심 가치
> "입력 3줄 → AI가 완성된 캐릭터 한 번에 생성 → 바로 게임 플레이"

---

## 3. 타겟 사용자

- RPG 게임을 즐기는 모바일 게이머
- 자신만의 캐릭터를 만들고 싶지만 그림이나 설계 능력이 없는 사람
- AI 기반 개인화 경험에 관심 있는 사용자

---

## 4. 핵심 기능 목록

### P0 — 반드시 구현 (MVP)

| 기능 | 설명 |
|------|------|
| 캐릭터 생성 입력 | 외형, 무기, 컨셉 텍스트 입력 화면 |
| AI 캐릭터 생성 | Claude API로 이름·스탯·능력·스토리 자동 생성 |
| 캐릭터 결과 화면 | 생성된 캐릭터 정보 표시 |
| Firebase 저장 | 생성된 캐릭터 Firestore에 저장 |
| 캐릭터 불러오기 | 저장된 캐릭터 목록 조회 및 선택 |
| 텍스트 기반 전투 | 생성된 캐릭터로 전투 플레이 |

### P1 — 구현 목표

| 기능 | 설명 |
|------|------|
| 캐릭터 삭제 | 저장된 캐릭터 삭제 |
| 전투 스킬 사용 | 특수능력 발동 로직 |
| 게임 오버 / 클리어 | 승패 화면 및 메인 메뉴 복귀 |

### P2 — 여유 있으면 구현

| 기능 | 설명 |
|------|------|
| 다중 스테이지 | 스테이지별 적 강도 증가 |
| 전투 기록 저장 | Firebase에 전투 결과 저장 |

## 5. 기술 스택 및 선정 이유

| 레이어 | 기술 | 선정 이유 |
|--------|------|----------|
| 게임 클라이언트 | Unity 2D | 모바일 게임 개발 표준, Android/iOS 동시 빌드 가능 |
| 백엔드 | Node.js + Express | 비동기 API 호출에 적합, Claude/Firebase 연동 간단 |
| AI 생성 | Claude API (claude-sonnet-4-6) | 한국어 이해 우수, JSON 응답 안정적, 속도·비용 균형 |
| 데이터베이스 | Firebase Firestore | 설정 간단, 무료 티어 충분, 실시간 동기화 |
| AI 개발 도구 | Claude Code | 코드·문서·커밋 자동화 |

---

## 6. 제약 조건 및 가정

- 이미지 없음 — 모든 게임 표현은 텍스트 UI로 처리
- 1인 사용자 기준 — 계정 시스템·멀티플레이 없음
- Claude API 비용 — 개발 중 mock 응답으로 비용 절감
- Firebase 무료 티어 사용 (읽기 50,000회/일, 쓰기 20,000회/일)
